Generate a grid

This test uses the @grid rule to create the grid. It then uses the classes, constants and mixins generated by this plugin to create a layout.

This is similar to the grid tests within Blueprint and 960.gs.

@grid
{
	column-width:50;
	column-count:12;
	gutter-width:10px;
	baseline:20px;
}

Mixins

Scaffold includes Mixins similar to that of SASS.

Mixins look like this:

=mixin-name(!param = #000, !param2)
{
	color:!param;
	height:!param2;
}

Or this:

=another
{
	text-decoration:none;
}

Then you assign them to a selector like a property:

#id { 
	+mixin-name(#fff, 100px); 
	+another; 
}

Constants

Constants can reduce the amount of reused code within your css. They are mostly useful for colors, but they can come in handy for other things, such as selector groups.

You set constants using @constants and set them like normal CSS properties. See the example below.

It's important to know that there is no logic involved in these constants. It doesn't matter where they are positioned in the css, as they are all pulled out at the start.

@constants
{
	body_background:#fff;
	body_color:#777;
	heading_color:#111;
}

Expressions

You can perform math and other expressions within your css very easily. Any property value which contains [expression] is parsed as php. See the examples below.

You CAN include CSS constants you've set using @constants.

#id
{
	padding:[(10*3) - 1]px;
}

Image replace

Image replacing content and text is extremely easy. Just use the image-replace property and use a url to the image.

The url is relative to the originally requested file. This is important to remember. If you get stuck, use an absolute path to the image.

.image-replacement
{
	image-replace:url(titles/none.png);
}

Multiple Properties

A simple plugin, this lets you use multiple properties to give the same property value.

.multiple-properties
{
	border, background-color, color: #000;
	border:1px solid;
}

.multiple-properties-2
{
	margin,padding:0;
	list-style,background:none;
}

Nested Selectors

You can nest selectors to clean up your code. It borrows the syntax from SASS for referencing the parent element also.

Using nested selectors isn't hard, and works just like you would expect.

Remember, Mixins can also include nested selectors.

#id
{
  height:200px;
  width:100px;

  a 
  {
    color:#fff;
    border-bottom:1px solid #eee;
    
    &:hover { border-color:#000; }
  }
}

Object Oriented CSS

You can extend a selector by using the property extends. This allows the selector to mimic the parent and extend its properties.

This allows you to avoid stacking classes in your markup.

.box 
{
	border:1px solid #eee;
	background:#fff;
	height:50px;
}

.special-box
{
	extends:'.box';
	padding:10px;
	border-width:4px;
	margin-top:!baseline;
}

Border-box Box Model

The W3C box model is annoying for creating layouts. The padding extends the width of the element, forcing you to recalculate the width of the element.

You can switch to the border-box method of box sizing by using the +border-box mixin. This applies the CSS3 property to browsers that support it, and uses a behaviour to get Internet Explorer to work correctly.

.box-sizing
{
	+border-box;
	padding:20px;
	float:left;
	width:210px;
}

Other features

CSScaffold also minifies, optimizes (with CSSTidy) and gzips your CSS before sending it to the browser.

If there are any features you need, but Scaffold doesn't provide, the plugin system makes it extremely easy to add your own functionality using the helper functions to navigate and manipulate the CSS.
